Conteúdo


Este trabalho prático teve como principal objetivo a generalização do conversor XML -> HTML desenvolvido nas aulas práticas, de forma a poder albergar vários relatórios (organizados num portefólio XML) e o índice respetivo.


Ora, para começar, criou-se dois ficheiros principais: um "portefolio.xml" (que contém a versão XML de todos os relatórios) e, subsequentemente, um "portefolio.dtd" (que contém as tags de todos os relatórios e a sua hierarquia). De forma a consultar estes ficheiros e deles extrair a informação necessária para gerar os relatórios em HTML, desenvolveram-se funções em Python, seguindo o raciocínio explicado de seguida.


Assim, abriu-se o ficheiro "portefolio.xml" em modo de leitura, usando a função read_file. Depois, extraiu-se e guardou-se numa lista as tags do ficheiro "portefolio.dtd", com a função extract_Dtd. Para maior organização dos ficheiros que iriam ser criados, criou-se uma pasta, onde foram guardados os relatórios em HTML e o índice (SE ela ainda não existisse). Mais ainda, criou-se um dicionário para guardar os títulos e hashtags de cada um dos relatórios, para, posteriormente, preencher o ficheiro "index.html"


Deste modo, fez-se um "split" do ficheiro XML, de forma a obter uma lista em que cada elemento é um relatório. Posto isto, para cada um dos relatórios, efetuou-se um conjunto de operações. Primeiramente, criou-se um dicionário {tag : miolo} recorrendo à função extract_Dict. De seguida, para as tags que, dentro delas, incluem outras ("authors", "description" e "references"), efetuou-se um processamento adicional: na tag "authors", estão incluídos vários "element", que foram extraídos para uma lista com a função extract_ListaH; para cada um desses "element", com a função extract_Dict, extraiu-se o miolo das tags "name", "number" e "email". Depois, para a tag "description", extraiu-se o conteúdo de cada tag "text" e "hashtag", e, para a tag "references", extraiu-se o conteúdo de cada tag "ref", usando, novamente, a função extract_ListaH. Finalmente, preencheu-se o template jinja generalizado para os relatórios (e previamente construído no ficheiro de código) com toda a informação extraída, para isso usando a função preenche_Report, e guardou-se cada um dos relatórios na pasta criada inicialmente.


Com vista a melhor aceder aos relatórios, criou-se também um template jinja para o índice, que foi preenchido com a informação contida no dicionário referido inicialmente (usando a função preenche_Indice) e guardou-se o ficheiro resultante ("index.html") na mesma pasta.
